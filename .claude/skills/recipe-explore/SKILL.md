# recipe-explore — 발상·후보 선정

## 할 일
1. `data.md`(또는 `assets/data.json`)에서 조건에 맞는 후보를 추린다.
   - 기본 우선순위: `ref=new` + 난이도 낮음 + 차별성 높음.
   - 타깃 지정 시(외국인·비건·시니어·건강·신세대) 해당 가중치 고려.
2. **선행사례 확인** (중요) — `new`로 밀 후보는 웹 검색으로 실제 레퍼런스를 확인한다.
   - 유사 사례가 있으면 → 차별점을 어떻게 잡을지 제안하고 `maybe`로 정정.
   - 검색 결과·출처를 남긴다(추측은 추측이라 표시).
3. 후보 3~5개를 한 줄 근거와 함께 제시. 사용자가 고르면 → `recipe-develop`로.

## 데이터 정합성
- 후보 상태(ref) 변경 시 `data.md` + `assets/data.json`(→`data.js` 재생성)을 함께 갱신.
